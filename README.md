**Security Incident Tracking System

** Overview
This project is a SQL-based system for tracking and analyzing security incidents to identify recurring patterns and prioritize remediation efforts.

** Objective
The goal of this project is to simulate how organizations track security incidents, analyze trends, and identify control gaps using structured data.

** Tools Used
- SQL

** Features
- Tracking of security incidents across systems
- Categorization of incident types and severity levels
- Identification of recurring issues and high-risk patterns

** Methodology
Incident data is structured and stored in tables, then queried using SQL to identify trends such as frequent incident types, high-risk assets, and recurring vulnerabilities.

** Outcome
This project demonstrates the ability to work with structured security data, perform analysis using SQL, and support risk-based decision-making in a cybersecurity context.
